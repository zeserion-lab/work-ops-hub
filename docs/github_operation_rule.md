# GitHub主軸運用ルール
## 業務 / 開発 / 私用 共通ルール文書

- 文書名: GitHub主軸運用ルール
- 版数: v1.0
- 作成日: 2026-04-14
- 目的: 業務・開発・私用における情報管理、タスク管理、履歴管理、自動化運用をGitHub主軸で統一し、探す時間の削減と再開しやすい運用を実現する

---

# 1. 目的

本ルールは、GitHubを主軸とした運用により、以下を実現することを目的とする。

- 情報探索時間の削減
- 次にやるべきことの明確化
- 判断経緯・変更履歴の記録
- AIで読み取りやすい構造化
- 中断後の再開容易化
- 業務 / 開発 / 私用の整理された共通運用

---

# 2. 適用範囲

本ルールは、以下3系統の管理対象に適用する。

## 2.1 業務
- 懸案事項
- 会議論点
- 要確認事項
- 横断プロジェクト
- 法令・品質・物流・在庫・請求・マスタ管理等

## 2.2 開発
- アプリ改修
- 不具合管理
- 仕様整理
- ドキュメント整備
- リリース管理
- テスト記録

## 2.3 私用
- 学習計画
- 資格取得
- 将来設計
- 旅行検討
- 個人改善テーマ
- 個人プロジェクト

---

# 3. 基本方針

## 3.1 全体方針
- GitHubを仕事と個人活動の共通管理基盤とする
- 業務 / 開発 / 私用は混在させすぎず、管理単位を分ける
- 書き方・見方・更新ルールは可能な限り統一する
- AI連携を前提とし、Issue・コメント・Summaryは簡潔かつ構造化する

## 3.2 管理単位
- 1テーマ = 1 Issue
- 1成果物 = 1 Repository
- 一覧管理 = Project
- 定期処理 = Actions
- 実行結果 = `logs/` または `summary/`

# 4. リポジトリ運用ルール

## 4.1 業務系リポジトリ
### 目的
- 懸案事項、会議論点、要確認事項、部門横断案件の管理

### 推奨例
- `work-ops-hub`
- `work-compliance`
- `work-meeting-notes`
- `work-logistics-projects`

### 主な格納内容
- Issues
- meeting summary
- 調査メモ
- 判断メモ
- AI要約結果
- 定期実行スクリプト

## 4.2 開発系リポジトリ
### 目的
- アプリ別の改善・不具合・仕様・リリース管理

### 推奨例
- `factory-audit-app`
- `manual-mvp`
- `rfid-order-app`
- `block-text-copy-beta`

### 主な格納内容
- ソースコード
- docs
- Issues
- Actions
- changelog
- test notes

## 4.3 私用系リポジトリ
### 目的
- 学習、計画、生活改善、個人プロジェクト管理

### 推奨例
- `personal-dashboard`
- `personal-study`
- `personal-plans`

### 主な格納内容
- 学習ログ
- 読書メモ
- 今週やること
- 将来計画
- AI用整理データ

# 5. Issue運用ルール

## 5.1 タイトル命名ルール
Issueタイトルは以下形式を基本とする。

- `[業務] RFIDタグ品番統一の確認`
- `[業務] 最新商品マスタ更新フロー整理`
- `[開発] RecordListPageの改善依頼対象フィルタ修正`
- `[私用] 統計検定2級 学習計画の見直し`
